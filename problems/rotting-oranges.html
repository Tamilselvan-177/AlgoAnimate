Rotting Oranges

Multi-source BFS. Every rotten orange spreads simultaneously — find the minimum minutes until no fresh orange remains, or return -1 if impossible.

Problem Statement

You are given an m × n grid where each cell is one of:

  • 0 — empty cell
  • 1 — fresh orange
  • 2 — rotten orange

Every minute, any fresh orange that is 4-directionally adjacent to a rotten orange becomes rotten.

Return the minimum number of minutes until no fresh orange remains. Return -1 if impossible.
Example 1
Input: [[2,1,1],[1,1,0],[0,1,1]]
Output: 4
Rotten spreads outward in 4 waves
Example 2
Input: [[2,1,1],[0,1,1],[1,0,1]]
Output: -1
Bottom-left orange is isolated — never reachable
Example 3
Input: [[0,2]]
Output: 0
No fresh oranges to begin with

Constraints

m == grid.length
n == grid[i].length
1 ≤ m, n ≤ 10
grid[i][j] is 0, 1, or 2

Key observations

  • Multiple rotten oranges spread simultaneously → multi-source BFS
  • Enqueue all rotten cells at minute 0
  • Process level by level — each level = 1 minute
  • After BFS, if fresh > 0 → return -1
  • No visited set needed — recoloring prevents revisit
Multi-Source BFS
Time O(m × n) Space O(m × n) 
from collections import deque

def orangesRotting(grid):
    rows, cols = len(grid), len(grid[0])
    q = deque()
    fresh = 0

    # Step 1: seed queue with all rotten, count fresh
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2: q.append((r, c, 0))  # (row, col, minute)
            if grid[r][c] == 1: fresh += 1

    # Step 2: early exit
    if fresh == 0: return 0

    # Step 3: multi-source BFS
    minutes = 0
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while q:
        r, c, t = q.popleft()
        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if (0 <= nr < rows and
                0 <= nc < cols and
                grid[nr][nc] == 1):
                grid[nr][nc] = 2           # rot it
                fresh -= 1
                minutes = t + 1
                q.append((nr, nc, t + 1))

    return minutes if fresh == 0 else -1
01
Why multi-source BFS?
All rotten oranges spread at the same time each minute. Starting BFS from one rotten cell at a time would give wrong answers. Enqueue all initial rotten cells at minute 0 so they all spread in parallel.
02
Level = 1 minute
Each BFS "level" represents one minute of spreading. Cells enqueued with timestamp t rot at minute t. Their neighbours rot at t+1. The maximum t across all rotted cells is the answer.
03
No visited set needed
We recolor fresh → rotten (1 → 2) immediately on enqueue. A rotten cell will never match grid[nr][nc] == 1 again, so it can never be re-enqueued. The grid itself is the visited set.
04
The -1 case
After BFS drains, any remaining fresh cells are unreachable — surrounded entirely by empty cells. Check fresh > 0 after the loop and return -1.
Key insight: The answer is the BFS distance from the nearest initial rotten orange to the farthest reachable fresh orange. Multi-source BFS computes all these shortest distances in a single O(m×n) pass — the same idea as "distance transform" in image processing.
