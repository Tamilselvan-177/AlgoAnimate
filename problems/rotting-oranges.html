Rotting Oranges

Multi-source BFS. Every rotten orange spreads simultaneously — find the minimum minutes until no fresh orange remains, or return -1 if impossible.

⏱ ~12 min to understand

Problem Statement

You are given an m × n grid where each cell is:

  0 — empty cell
  1 — fresh orange
  2 — rotten orange

Every minute, any fresh orange 4-directionally adjacent to a rotten orange becomes rotten.

Return the minimum number of minutes until no fresh orange remains. Return -1 if impossible.
Example 1 — output: 4
Input: [[2,1,1],[1,1,0],[0,1,1]]
Output: 4
Rotten spreads outward in 4 waves
Example 2 — output: -1
Input: [[2,1,1],[0,1,1],[1,0,1]]
Output: -1
Bottom-left orange is isolated
Example 3 — output: 0
Input: [[0,2]]
Output: 0
No fresh oranges at start

Constraints

m == grid.length
n == grid[i].length
1 ≤ m, n ≤ 10
grid[i][j] ∈ {0, 1, 2}
Complexity
Time: O(m × n)
Space: O(m × n)
Every cell is visited at most once

Key observations

Multiple rotten oranges spread simultaneously → multi-source BFS.

Enqueue all initial rotten cells at minute 0.

After BFS, if fresh > 0 → isolated orange → return -1.
Multi-Source BFS
O(m × n)O(m × n) space
from collections import deque

def orangesRotting(grid):
    rows, cols = len(grid), len(grid[0])
    q = deque()
    fresh = 0

    # Step 1: seed queue with ALL rotten, count fresh
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == 2: q.append((r, c, 0))  # (row, col, minute)
            if grid[r][c] == 1: fresh += 1

    # Step 2: early exit — no fresh oranges
    if fresh == 0: return 0

    # Step 3: multi-source BFS
    minutes = 0
    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while q:
        r, c, t = q.popleft()
        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if (0 <= nr < rows and
                0 <= nc < cols and
                grid[nr][nc] == 1):
                grid[nr][nc] = 2           # rot it immediately
                fresh -= 1
                minutes = t + 1
                q.append((nr, nc, t + 1))

    return minutes if fresh == 0 else -1
💡 BFS level = 1 minute. Enqueue all rotten cells at t=0 so they spread in lockstep — that's what "simultaneously" means in code.

Step 1 — Why multi-source BFS?

The problem says rotten oranges spread simultaneously. If you ran single-source BFS from one rotten orange, then another, you'd compute the wrong times — later sources might reach a cell earlier than your first BFS said.
❌ Wrong — sequential BFS
Run BFS from source A, then from source B.
Cell X might be found at t=3 from A, but
t=1 from B — you'd record the wrong time.
✅ Correct — multi-source BFS
Enqueue ALL sources at t=0 together.
BFS naturally finds the shortest distance
from the nearest rotten source to each cell.
Think of it like: adding a virtual super-source node connected to all initial rotten cells at cost 0. BFS from the super-source gives the correct minimum-time spreading.

Step 2 — BFS level = 1 minute

Each BFS "level" represents exactly one minute of real-world time. We store the timestamp t directly in the queue entry (row, col, t).
How timestamps flow through the queue
t=0  initial rotten cells → enqueued at minute 0
t=1  their fresh neighbours → rot at minute 1
t=2  neighbours of those → rot at minute 2
The final answer = max t across all cells that rotted
minutes = t + 1 is updated every time a fresh orange rots. After the loop, minutes holds the last time any orange rotted — which is the answer.

Step 3 — No visited set needed (the grid IS the visited set)

Standard BFS needs a visited set to prevent revisiting. Here we get that for free:
The trick
We immediately set grid[nr][nc] = 2 on enqueue — before it's even processed. So the check grid[nr][nc] == 1 will never be true again for that cell.
Why it's safe
Rotten cells (2) and empty cells (0) are never enqueued. Only fresh (1) → rotten (2) transitions happen. Recoloring on enqueue = O(1) visited tracking.
Compare to standard BFS visited pattern Standard BFS: visited.add(node) before enqueue
This problem: grid[nr][nc] = 2 before enqueue — same effect, zero overhead

Step 4 — The -1 case: isolated oranges

After BFS drains completely, check if fresh > 0. If yes, those oranges are completely surrounded by empty cells — no rotten orange can ever reach them.
Example that returns -1: [[2,1,1],[0,1,1],[1,0,1]]
Rotten: (0,0) only
Reachable fresh: (0,1),(0,2),(1,1),(1,2) → all rot by t=4
Unreachable: (2,0) — blocked by empty cell (1,0) and (2,1)
fresh = 1 after BFS → return -1
Always count fresh before BFS and decrement on every rotting — final check is O(1), not a second grid scan.

Step 5 — Edge cases that trip people up

❌ No fresh from the start
If fresh == 0 initially, return 0 immediately — even if the queue is empty. Don't fall through to the BFS loop.
❌ No rotten oranges
Queue is empty, BFS loop never runs, minutes = 0. If fresh > 0 we correctly return -1. The pattern handles it without special casing.
❌ minutes stays 0
minutes is only updated when a fresh orange rots. If all fresh oranges were already adjacent to rotten at t=0, minutes correctly stays 0 after the early-exit check.
❌ Mutating input grid
This solution mutates the grid. If the problem requires non-destructive, deep-copy the grid first or use a separate visited array.
The mental model: Think of rotting as water flooding — all sources release water simultaneously at t=0. The answer is how many seconds until the last dry cell gets wet (or -1 if some cells are in sealed pockets). Multi-source BFS is exactly "flood fill from multiple starting points at the same time."
💡 Multi-source BFS = flood fill from all rotten oranges simultaneously. Watch the queue grow level by level.
