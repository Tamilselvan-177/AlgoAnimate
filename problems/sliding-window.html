Technique #01

Sliding Window

Maintain a moving subarray to solve subarray/substring problems in O(n) — instead of the brute-force O(n²) nested loop approach.

Problem Statement

Max Sum Subarray of Size K

Given an array of integers and a number k, find the maximum sum of any contiguous subarray of size k.

Variant 2: Longest substring without repeating characters — find the length of the longest substring with all unique characters.
Example 1 — Fixed Window
Input: arr = [2,1,5,1,3,2], k = 3
Output: 9 (subarray [2,1,5] → wait, [1,5,1]? No — [5,1,3] = 9)
Example 2 — Dynamic Window
Input: s = "abcabcbb"
Output: 3 (substring "abc")

Constraints

1 ≤ arr.length ≤ 10⁵
-10⁴ ≤ arr[i] ≤ 10⁴
1 ≤ k ≤ arr.length
s consists of printable ASCII characters

When to use Sliding Window

  • Problem asks for a subarray / substring
  • Involves contiguous elements
  • Asks for max / min / longest / shortest
  • Has a clear expand / shrink condition

Max Sum Subarray of Size K
Time O(n) Space O(1) 
def max_sum_subarray(arr, k):
    n = len(arr)
    if n < k:
        return -1

    # Compute sum of first window
    window_sum = sum(arr[:k])
    best = window_sum

    # Slide the window: add right, drop left
    for R in range(k, n):
        window_sum += arr[R]       # incoming element
        window_sum -= arr[R - k]   # outgoing element
        best = max(best, window_sum)

    return best
Longest Substring Without Repeating Characters
Time O(n) Space O(k) 
def longest_no_repeat(s):
    seen = {}  # char → last seen index
    L = 0
    best = 0

    for R in range(len(s)):
        # If duplicate inside window, shrink from left
        if s[R] in seen and seen[s[R]] >= L:
            L = seen[s[R]] + 1

        seen[s[R]] = R  # record latest position
        best = max(best, R - L + 1)

    return best
01
The core idea
Instead of summing every subarray from scratch (O(n²)), keep a running total. Slide one step: add the new element on the right, drop the old one on the left. One pass = O(n).
02
Two pointers L and R
R always moves forward every iteration. L only moves forward when a constraint is violated — like a duplicate character entering the window. This asymmetry is what keeps it O(n).
03
Fixed vs dynamic
Fixed window: size k is constant, both L and R move together. Dynamic window: size changes — R expands freely, L catches up only when the window breaks a rule.
04
Universal template
for R in range(n): expand window. while window invalid: shrink (L++). update answer. This template solves nearly every sliding window problem.
Key insight: Every element enters the window exactly once (when R reaches it) and leaves exactly once (when L passes it). So the total work across all iterations is O(n) — not O(n²) even though there's a nested while loop.
