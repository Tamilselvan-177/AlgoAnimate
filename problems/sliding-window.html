Technique #01

Sliding Window

Maintain a moving subarray to solve subarray/substring problems in O(n) — instead of the brute-force O(n²) nested loop approach.

⏱ ~12 min to understand

Problem Statement

Variant 1 — Max Sum Subarray of Size K

Given an array of integers and a number k, find the maximum sum of any contiguous subarray of exactly size k.

Variant 2 — Longest Substring Without Repeating Characters

Find the length of the longest substring where all characters are unique.
Example 1 — Fixed Window
arr = [2,1,5,1,3,2], k = 3
Output: 9   →  subarray [5,1,3]
Example 2 — Dynamic Window
s = "abcabcbb"
Output: 3   →  substring "abc"

Constraints

1 ≤ arr.length ≤ 10⁵
-10⁴ ≤ arr[i] ≤ 10⁴
1 ≤ k ≤ arr.length
s consists of printable ASCII

When to use Sliding Window

Problem asks for a subarray / substring and involves contiguous elements.

Max Sum Subarray of Size K
O(n)O(1) space
def max_sum_subarray(arr, k):
    n = len(arr)
    if n < k: return -1

    # Compute sum of first window
    window_sum = sum(arr[:k])
    best = window_sum

    # Slide: add right element, drop left element
    for R in range(k, n):
        window_sum += arr[R]        # incoming
        window_sum -= arr[R - k]    # outgoing
        best = max(best, window_sum)

    return best
Longest Substring Without Repeating Characters
O(n)O(k) space
def longest_no_repeat(s):
    seen = {}  # char → last seen index
    L = 0
    best = 0

    for R in range(len(s)):
        # Duplicate inside window → shrink from left
        if s[R] in seen and seen[s[R]] >= L:
            L = seen[s[R]] + 1

        seen[s[R]] = R          # record latest position
        best = max(best, R - L + 1)

    return best
💡 Every element enters the window once and leaves once. Total work = O(n), never O(n²) — even with a nested shrink loop.

Step 1 — The core idea: slide, don't restart

Instead of summing every subarray from scratch (O(n²)), keep a running total and adjust it by one element at a time.
❌ Brute force O(n²)
For each start position, loop through k elements and sum them. O(n × k) = O(n²) for large k.
✅ Sliding Window O(n)
Slide one step: add arr[R], drop arr[R-k]. One arithmetic op per step — O(n) total.

Step 2 — Two pointers L and R

R always moves forward every iteration. L only moves forward when a constraint is violated.
R expands → always, every step
L shrinks → only when window is invalid (duplicate found, size exceeded, etc.)
This asymmetry is what keeps the overall loop O(n) — L never goes backwards.

Step 3 — Fixed vs dynamic window

Fixed window (size k)
L and R always stay k-1 apart. Both move together — drop one left, add one right.
Dynamic window (variable)
Size changes. R expands freely, L catches up only when the window breaks a rule.

Step 4 — Universal template

def sliding_window(arr):
    L = 0

This template solves nearly every sliding window problem. The shrink loop looks O(n²) but it's actually O(n) amortised — L only ever moves right.
Key insight: Every element enters the window exactly once (when R reaches it) and leaves exactly once (when L passes it). Total work across all iterations = O(n) — not O(n²) even with a nested while loop.
