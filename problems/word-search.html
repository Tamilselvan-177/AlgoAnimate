Word Search

Given a 2D grid of characters, determine if a given word exists by following a path of adjacent cells. Each cell may only be used once. The classic backtracking on a grid.

⏱ ~12 min

Problem Statement

Given an m × n grid of characters board and a string word, return true if word exists in the grid.

Word must be constructed from letters of sequentially adjacent cells (horiz/vert). The same cell may not be used more than once.
Example 1
board = [["A","B","C","E"],["S","F","C","S"],["A","D","E","E"]]
word = "ABCCED" → true
word = "SEE" → true
word = "ABCB" → false (can't revisit B)

Constraints

1 ≤ m, n ≤ 6
1 ≤ word.length ≤ 15
Uppercase English letters only
Complexity
Time: O(m·n·4L)
Space: O(L) recursion depth
In-place marking
Explicit visited
Backtracking — mark in-place

def exist(board, word):
    m, n = len(board), len(board[0])
    def dfs(r, c, idx):
        if idx == len(word): return True
        if r < 0 or r >= m or c < 0 or c >= n: return False
        if board[r][c] != word[idx]: return False
        tmp = board[r][c]
        board[r][c] = '#'           # mark visited
        found = (dfs(r+1,c,idx+1) or dfs(r-1,c,idx+1)
              or dfs(r,c+1,idx+1) or dfs(r,c-1,idx+1))
        board[r][c] = tmp             # restore (backtrack)
        return found
    return any(dfs(r,c,0) for r in range(m) for c in range(n))

def exist(board, word):
    m, n = len(board), len(board[0])
    vis = [[False]*n for _ in range(m)]
    def dfs(r, c, idx):
        if idx == len(word): return True
        if not (0 <= r < m and 0 <= c < n): return False
        if vis[r][c] or board[r][c] != word[idx]: return False
        vis[r][c] = True
        found = any(dfs(r+dr,c+dc,idx+1) for dr,dc in [(1,0),(-1,0),(0,1),(0,-1)])
        vis[r][c] = False
        return found
    return any(dfs(r,c,0) for r in range(m) for c in range(n))

Step 1 — Try every cell as start

We don't know where the word starts, so we try every cell. For each cell that matches word[0], launch a DFS.

Step 2 — Three base cases

✓ Found
idx == len(word)
All chars matched
✗ OOB
r/c out of bounds
Left the grid
✗ Mismatch
board[r][c] ≠ word[idx]
Wrong char or visited

Step 3 — Mark → recurse → unmark

tmp = board[r][c]
board[r][c] = '#'  # mark — blocks revisit
found = dfs(down) or dfs(up) or dfs(right) or dfs(left)
board[r][c] = tmp     # restore ← backtrack

Step 4 — Full trace: word = "SEE" in Example 1 grid

Scan for 'S'… found at (1,0) and (1,3)
Start (1,0): board[1][0]='S' ✓ → try neighbours for 'E'
(0,0)='A' ✗   (2,0)='A' ✗   (1,1)='F' ✗ → all fail from (1,0)
Start (1,3): board[1][3]='S' ✓ → try neighbours for 'E'
(0,3)='E' ✓ → mark '#', try neighbours for second 'E'
(0,2)='C' ✗   (1,3)='#' ✗ (visited)   up = OOB → all fail
↩ backtrack — restore (0,3)='E'
(2,3)='E' ✓ → mark '#', try neighbours for second 'E'
(1,3)='#' ✗ (visited)   (2,2)='E' ✓ → idx=3 == len("SEE")
→ return True ✓ — path: (1,3) → (2,3) → (2,2)

Step 5 — Common mistakes

✗ Not restoring board
Forgetting board[r][c] = tmp corrupts the board for future DFS calls starting from other cells.
✗ Wrong base case order
Check idx == len(word) BEFORE bounds check — otherwise a word that ends exactly at the edge returns False.
✗ Revisiting same cell
Without marking visited, "AAAAAA" on a grid of A's would incorrectly reuse the same cell multiple times.
✗ Only trying first match
Must try every cell as a start. The first cell matching word[0] might not lead to the full word.

DFS call tree — word = "SEE", board = ABCE / SFCS / ADEE

Solid edge = recurse deeper  ·  Dashed edge = pruned (mismatch)  ·  ● exploring   ● matched ✓   ● pruned ✗
Starting from (1,3)='S': we find 'E' at both (0,3) and (2,3). From (0,3) no second 'E' is reachable. From (2,3), cell (2,2)='E' completes the word — SEE found!
Pattern: mark → recurse → unmark. The same template works for Sudoku, N-Queens, and permutations.
