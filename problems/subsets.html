Subsets

Generate all 2ⁿ subsets of an array of unique integers. The classic backtracking / power set problem — every element is either included or excluded at each level of the recursion tree.

Problem Statement

Given an integer array nums of unique elements, return all possible subsets (the power set).

The solution must not contain duplicate subsets. Return in any order.

A subset is any selection of 0 or more elements — including the empty set [] and the full array itself.
Example 1
Input: nums = [1,2,3]
Output: [[],[1],[2],[1,2],[3],[1,3],[2,3],[1,2,3]]
8 subsets = 2³
Example 2
Input: nums = [0]
Output: [[],[0]]
2 subsets = 2¹

Constraints & Key Facts

1 ≤ nums.length ≤ 10
-10 ≤ nums[i] ≤ 10
All numbers are unique
Output sizes

def subsets(nums):
    res = []

    def backtrack(start, current):
        res.append(current[:])   # record at EVERY call
        for i in range(start, len(nums)):
            current.append(nums[i])   # include
            backtrack(i + 1, current) # explore
            current.pop()             # exclude (backtrack)

    backtrack(0, [])
    return res
Iterative Cascade
O(n × 2ⁿ)O(1) extra
def subsets(nums):
    res = [[]]
    for num in nums:
        res += [sub + [num] for sub in res]
    return res

Bit Manipulation
O(n × 2ⁿ)O(1) extra
def subsets(nums):
    n, res = len(nums), []
    for mask in range(1 << n):
        res.append([nums[i] for i in range(n) if mask & (1 << i)])
    return res

# mask 000 → []    mask 001 → [1]
# mask 010 → [2]   mask 011 → [1,2]
# mask 100 → [3]   mask 111 → [1,2,3]

Step 1 — Every element has exactly 2 choices

For each element in nums, you make one binary decision: include it or skip it. With n independent binary choices, the total number of outcomes is 2 × 2 × … × 2 = 2ⁿ.

✗
Skip

For n=3: 2³ = 8 subsets. For n=10: 2¹⁰ = 1024 subsets. This is called the power set.

Step 2 — What backtracking does, step by step

Backtracking = try → recurse deeper → undo → try next option. We mutate a single current list because undoing is just pop().
Full trace for nums=[1,2,3]
call backtrack(0, []) → record []
call backtrack(1, [1]) → record [1]
call backtrack(2, [1,2]) → record [1,2]
call backtrack(3, [1,2,3]) → record [1,2,3]
pop(3) → cur=[1,2]
pop(2) → cur=[1]
call backtrack(3, [1,3]) → record [1,3]
pop(3) → cur=[1]
pop(1) → cur=[]
call backtrack(2, [2]) → record [2]
call backtrack(3, [2,3]) → record [2,3]
pop(3) → cur=[2]
pop(2) → cur=[]
call backtrack(3, [3]) → record [3]
pop(3) → cur=[]
pop() is the "undo". After every recursive call returns, we remove what we added. One array is reused for the entire search — no copying needed until we record a result.

Step 3 — Why the start index prevents duplicates

Without start, we'd revisit earlier elements and produce [1,2] AND [2,1] — two representations of the same subset.
❌ Without start
for i in range(0, n):
generates [1,2] AND [2,1]
generates [1,3] AND [3,1]
✓ With start
for i in range(start, n):
only looks FORWARD
[1,2] only, never [2,1]
When we include nums[i] and recurse, we pass i+1 as new start. Subsets are always built in increasing index order — uniqueness is guaranteed structurally.

Step 4 — Record at EVERY call, not just at leaves

This is what makes Subsets different from all other backtracking problems. Every state of current[] is a valid subset — even an empty [] at the root.
