Trapping Rain Water

Treat each value as a vertical wall; rain fills the gaps between them. Each column has width 1. Return the total units of water trapped. At index i, depth is max(0, min(maxLeft, maxRight) − height[i]) — the surface height is min(maxLeft, maxRight), minus the bar.

⏱ ~15 min

Problem Statement

Given n non-negative integers representing an elevation map where the width of each bar is 1, return the total amount of water the map can hold after raining. Think of the array as a row of walls; water sits in valleys between taller neighbors.

At index i, water is bounded by the tallest bar strictly to the left and the tallest bar strictly to the right. Units above the bar at i equal min(maxLeft, maxRight) − height[i], or 0 if that value is negative. The first and last positions never trap water; with n < 3 the answer is always 0.
Example 1
height = [0,1,0,2,1,0,1,3,2,1,2,1]
Output: 6
Example 2
height = [4,2,0,3,2,5]
Output: 9

def trap(height):
    l, r = 0, len(height) - 1
    max_l = max_r = water = 0
    while l < r:
        if height[l] <= height[r]:
            if height[l] >= max_l:
                max_l = height[l]        # new left wall
            else:
                water += max_l - height[l]  # trapped!
            l += 1
        else:
            if height[r] >= max_r:
                max_r = height[r]        # new right wall
            else:
                water += max_r - height[r]  # trapped!
            r -= 1
    return water

O(n)O(n)
def trap(height):
    n = len(height)
    if n == 0: return 0
    max_l = [0] * n           # max height to the left of i
    max_r = [0] * n           # max height to the right of i
    max_l[0] = height[0]
    for i in range(1, n):
        max_l[i] = max(max_l[i-1], height[i])
    max_r[n-1] = height[n-1]
    for i in range(n-2, -1, -1):
        max_r[i] = max(max_r[i+1], height[i])
    water = 0
    for i in range(n):
        water += max(min(max_l[i], max_r[i]) - height[i], 0)
    return water
Monotonic Stack — horizontal layers
O(n)O(n)
def trap(height):
    stack, water = [], 0
    for r in range(len(height)):
        while stack and height[r] > height[stack[-1]]:
            bot = stack.pop()           # the "valley" bottom
            if not stack: break
            l = stack[-1]
            h = min(height[l], height[r]) - height[bot]
            w = r - l - 1
            water += h * w              # horizontal slab
        stack.append(r)
    return water

Step 1 — The water formula

At each index i, imagine a horizontal water surface: it cannot rise above the shorter of the highest wall on the left and the highest wall on the right. The surface height is min(maxLeft[i], maxRight[i]); subtract height[i] to get how many units sit above that bar (use max(0, …) on peaks).

water[i] = max(0, min(maxLeft[i], maxRight[i]) − height[i])

This is the foundation of all three approaches. They differ only in how they compute maxLeft and maxRight. In the DP solution those are full prefix/suffix arrays. In the two-pointer code, max_l and max_r are running maxima seen so far from each end — not the same symbols as maxLeft[i], but they carry exactly the information needed when you process from the shorter side.

Step 2 — Why two pointers work

Invariant: indices strictly outside [l, r] are finished; all water there is already accounted for. At each step we compare height[l] and height[r]. The shorter side is safe to resolve: the opposite side still has a wall at least that tall, so the true bottleneck for that column is known. We never move both pointers at once.
If height[l] ≤ height[r]
The right wall is at least as tall as the left. So maxLeft is the bottleneck. We know exactly how much water is at l. Advance l inward.
If height[r] < height[l]
The left wall is taller. maxRight is the bottleneck. We know exactly how much water is at r. Advance r inward.

Other solutions: DP precomputes maxLeft[i] and maxRight[i] then sums the same formula. The monotonic stack adds water in horizontal slabs between indices—same total, different order of aggregation. Pick the story that clicks for you.

Step 4 — Common mistakes

✗ Wrong pointer movement
Moving both pointers or always advancing l breaks the invariant. Only advance the side with the shorter max wall.
✗ Negative water
On a peak, min(max_l,max_r) − h[i] can be negative. Always clamp: max(0, ...) or the two-pointer approach never adds negatives.
✗ Off-by-one in stack
Width = r - l - 1, not r - l. The two walls themselves don't hold water.
✗ Single-element edge
An array of length 0 or 1 holds no water. The loop condition l < r handles this automatically.
Pattern: The two-pointer trick works because we always know the minimum of one side — whichever max is smaller. This "I can commit to this side" reasoning appears in container problems (LC 11) and many greedy height problems.

def trap(height):
  l, r = 0, len(height)-1
  max_l = max_r = water = 0
  while l < r:
    if height[l] <= height[r]:  # left side
      if height[l] >= max_l: max_l = height[l]
      else: water += max_l - height[l]
      l += 1
    else:                       # right side
      if height[r] >= max_r: max_r = height[r]
      else: water += max_r - height[r]
      r -= 1
  return water
