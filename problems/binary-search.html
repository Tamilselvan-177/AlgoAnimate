Advanced Binary Search

5 patterns — Classic, Lower Bound, Upper Bound, Binary Search on Answer Space (🔥 most important), and Peak Element. Custom input for every pattern.

What is Binary Search?

Binary Search eliminates half the search space in every step — giving O(log n) instead of O(n).

But advanced Binary Search goes beyond finding a value. We use it to:
  • Find first / last occurrence of a value
  • Find the minimum / maximum answer to an optimization problem
  • Find a peak element without a sorted array

Golden rule: If the problem has a monotonic condition (False→True or True→False), Binary Search works.
Monotonic property (key idea)
speed too slow? → False False False True True True ← fast enough
Binary Search finds the boundary between False and True

Classic Binary Search
Time O(log n)Space O(1)
def binarySearch(arr, target):
    left, right = 0, len(arr) - 1

    while left <= right:          # note: <=
        mid = (left + right) // 2

        if arr[mid] == target:
            return mid            # found!
        elif arr[mid] < target:
            left = mid + 1         # go right
        else:
            right = mid - 1        # go left

    return -1                      # not found
Lower Bound — first index ≥ target
Time O(log n)Space O(1)
def lowerBound(arr, target):
    left, right = 0, len(arr)    # right = len (not len-1)

    while left < right:           # note: strict <
        mid = (left + right) // 2

        if arr[mid] < target:
            left = mid + 1         # too small → go right
        else:
            right = mid             # candidate! shrink right

    return left                    # first i where arr[i] >= target

Upper Bound — first index > target
Time O(log n)Space O(1)
def upperBound(arr, target):
    left, right = 0, len(arr)

    while left < right:
        mid = (left + right) // 2

        if arr[mid] <= target:       # only change from lowerBound
            left = mid + 1         # push PAST equal values
        else:
            right = mid

    return left                    # first i where arr[i] > target

Binary Search on Answer — Koko Eating Bananas (LeetCode 875) 🔥
Time O(n log M)Space O(1)
import math

def minEatingSpeed(piles, h):
    def canEat(speed):
        # Check if we can eat all piles in h hours at this speed
        hours = sum(math.ceil(p / speed) for p in piles)
        return hours <= h

    left  = 1           # minimum possible speed
    right = max(piles)  # maximum possible speed

    while left < right:
        mid = (left + right) // 2

        if canEat(mid):
            right = mid        # feasible → try smaller (minimize)
        else:
            left = mid + 1    # too slow → must go faster

    return left  # minimum feasible speed

Peak Element — LeetCode 162
Time O(log n)Space O(1)
def findPeakElement(arr):
    left, right = 0, len(arr) - 1

    while left < right:
        mid = (left + right) // 2

        if arr[mid] < arr[mid + 1]:
            # ascending slope → peak must be to the RIGHT
            left = mid + 1
        else:
            # descending → peak is at mid or to the LEFT
            right = mid

    return left  # index of a peak element
L≤R
Classic condition
Use left <= right when searching for an exact value. mid = (L+R)//2 every step. Three outcomes: found, go right (L=mid+1), go left (R=mid-1).
L<R
Bound condition
Lower & upper bound use left < right. right starts at len(arr) not len-1. Loop terminates when L==R — that's the answer.
≥ vs >
Lower vs Upper
Lower bound: condition is arr[mid] < target → go right. Upper bound flips to arr[mid] <= target → go right (pushes past all equals).
🔥
Answer space trick
Don't search the array — search the answer range [1, max]. For each mid, run a feasibility check O(n). Total: O(n log M). Finds minimum valid answer.
∧
Peak: slope trick
Compare arr[mid] vs arr[mid+1]. If ascending → peak is right. If descending → peak is here or left. Doesn't need a sorted array — just needs a peak to exist.
F→T
Monotonic key
All answer-space problems share this: there's a threshold where condition flips from False to True. Binary search finds that threshold in O(log M) steps.
How to identify Binary Search on Answer Space: Problem says "minimum possible X", "maximum possible Y", "can we do this in K steps" — and there's a clear feasibility check function. If you can write canDo(mid) → True/False and it's monotonic, use this pattern.
