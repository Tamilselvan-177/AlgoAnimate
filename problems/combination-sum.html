Combination Sum

Find all unique combinations of candidates that sum to a target. Each number can be used unlimited times. The key difference from Subsets — repetition is allowed.

⏱ ~12 min to understand

Problem Statement

Given an array of distinct integers candidates and a target integer, return all unique combinations where the chosen numbers sum to target.

The same number may be chosen unlimited times. Two combinations are unique if their frequency of chosen numbers differs.

Return the answer in any order.
Example 1
Input: candidates=[2,3,6,7], target=7
Output: [[2,2,3],[7]]
2+2+3=7 and 7=7
Example 2
Input: candidates=[2,3,5], target=8
Output: [[2,2,2,2],[2,3,3],[3,5]]
Example 3
Input: candidates=[2], target=1
Output: []
No valid combination exists

Constraints

1 ≤ candidates.length ≤ 30
2 ≤ candidates[i] ≤ 40
All candidates are distinct
1 ≤ target ≤ 40

Subsets vs Combination Sum

  • Subsets: record every call, no repetition
  • Comb Sum: record only at remaining=0
  • Comb Sum: start stays at i (not i+1) to allow reuse
  • Comb Sum: prune when remaining < 0
Why sort first?
Sort candidates → enables pruning. Once a candidate exceeds remaining, all larger candidates will too. Stop early.

def combinationSum(candidates, target):
    candidates.sort()  # sort enables pruning
    res = []

    def backtrack(start, current, remaining):
        if remaining == 0:          # ← found a valid combination!
            res.append(current[:])
            return

        for i in range(start, len(candidates)):
            if candidates[i] > remaining:
                break              # ← prune: sorted, so rest are also too big
            current.append(candidates[i])
            backtrack(i, current, remaining - candidates[i])  # i not i+1 → reuse allowed
            current.pop()

    backtrack(0, [], target)
    return res

def combinationSum(candidates, target):
    res = []

    def backtrack(start, current, remaining):
        if remaining == 0:
            res.append(current[:])
            return
        if remaining < 0:  # no break, check each one
            return

        for i in range(start, len(candidates)):
            current.append(candidates[i])
            backtrack(i, current, remaining - candidates[i])
            current.pop()

    backtrack(0, [], target)
    return res

# Without sorting, we can't use break — must check every branch.
# Sorting + break eliminates entire subtrees → much faster on large inputs.

Step 1 — How this differs from Subsets

In Subsets, each element is used at most once. In Combination Sum, each candidate can be used unlimited times. This changes exactly one thing in the backtracking template.
Subsets (no reuse)
backtrack(i+1, cur)
↑ move forward, skip i
Combination Sum (reuse)
backtrack(i, cur)
↑ stay at i, can pick again
Also different: we only record when remaining == 0 (not at every call). Intermediate states are not valid answers here.

Step 2 — The pruning trick (why sort first)

If we sort candidates first, we can break early the moment a candidate exceeds remaining. Since the array is sorted, every candidate after it is also too large — no need to check them.
candidates=[2,3,6,7], target=4, current=[2,2]
remaining = 4-2-2 = 0   ✓ record [2,2]
Try 3: remaining=0-3=-3  ✗ 3 > 0 → break
Skip 6, skip 7 — not even tried!
Without pruning: O(N^(T/M)). With pruning: many branches cut. For target=40 and min=2, that's ~2^20 = 1M nodes vs far fewer with pruning.

Step 4 — The 3-change recipe vs Subsets

What changes Subsets Combination Sum
When to record Every call (top) Only when rem==0
Recurse with i+1 (no reuse) i (reuse allowed)
Prune when Never (explore all) candidate > remaining
Pattern to memorize: sort + break + backtrack(i, cur, rem-c[i])

The i (not i+1) in the recursive call is the single difference that allows unlimited reuse. Everything else — the for loop, append/pop, start index — is identical to Subsets.
