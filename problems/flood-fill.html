Flood Fill

The paint-bucket algorithm. Starting from pixel (sr, sc), repaint all 4-directionally connected pixels sharing the same original color with a new color.

Problem Statement

You are given an image represented by an m × n grid of integers, where image[i][j] is the pixel value.

You are also given three integers sr, sc, and color.

Perform a flood fill starting from pixel (sr, sc):
  • Change (sr, sc) to color
  • For each adjacent pixel (4-directional) that has the same original color as the start pixel — change it too
  • Repeat recursively for all newly changed pixels
  • Stop when no more adjacent pixels match the original color

Return the modified image.
Example 1
Input: image=[[1,1,1],[1,1,0],[1,0,1]], sr=1, sc=1, color=2
Output: [[2,2,2],[2,2,0],[2,0,1]]
Center pixel and all 1s connected to it become 2. Bottom-right 1 is not connected.
Example 2
Input: image=[[0,0,0],[0,0,0]], sr=0, sc=0, color=0
Output: [[0,0,0],[0,0,0]]
Start color already equals new color → return immediately, no change.

Constraints

m == image.length
n == image[i].length
1 ≤ m, n ≤ 50
0 ≤ image[i][j], color < 2¹⁶
0 ≤ sr < m
0 ≤ sc < n

Key observations

  • If origColor == color, return immediately — infinite loop risk
  • Recolor on enqueue — cell won't match origColor again, so no visited set needed
  • Only 4 directions (up/down/left/right), not diagonal
  • BFS and DFS both work — BFS shown here

BFS — Iterative
Time O(m × n) Space O(m × n) 
from collections import deque

def floodFill(image, sr, sc, color):
    origColor = image[sr][sc]

    # Edge case: already the target color
    if origColor == color:
        return image

    rows, cols = len(image), len(image[0])
    q = deque([(sr, sc)])
    image[sr][sc] = color          # paint on enqueue

    dirs = [(1,0), (-1,0), (0,1), (0,-1)]

    while q:
        r, c = q.popleft()
        for dr, dc in dirs:
            nr, nc = r + dr, c + dc
            if (0 <= nr < rows and
                0 <= nc < cols and
                image[nr][nc] == origColor):
                image[nr][nc] = color  # paint on enqueue
                q.append((nr, nc))

    return image
DFS — Recursive
Time O(m × n) Space O(m × n) 
def floodFill(image, sr, sc, color):
    origColor = image[sr][sc]
    if origColor == color: return image

    def dfs(r, c):
        if (r < 0 or r >= len(image) or
            c < 0 or c >= len(image[0]) or
            image[r][c] != origColor):
            return
        image[r][c] = color
        dfs(r+1, c); dfs(r-1, c)
        dfs(r, c+1); dfs(r, c-1)

    dfs(sr, sc)
    return image
01
Paint bucket analogy
Exactly like the bucket fill tool in MS Paint. Click a pixel → every touching pixel of the same color gets repainted. Stops at color boundaries.
02
No visited set needed
We recolor the cell immediately when we enqueue it — before processing. Since the cell no longer equals origColor, it can never be re-enqueued. The image itself is the visited set.
03
Critical edge case
If origColor == color, return immediately. Without this check, BFS would loop forever re-enqueuing the same cells (since their color always matches origColor after repainting).
04
4-directional only
Only up, down, left, right — not diagonal. The bottom-right corner in Example 1 stays unchanged because it's only connected diagonally to the flood region.
Key insight: Flood Fill is just BFS/DFS with a color-match condition instead of a visited-set condition. The recolor-on-enqueue trick is the same trick used in Rotting Oranges — the grid mutation doubles as the visited marker.
