Gray Code

Generate a sequence of 2ⁿ integers where every adjacent pair differs by exactly one bit. The elegant reflected construction builds each level by mirroring the previous — no backtracking, no search.

⏱ ~10 min to understand

Problem Statement

An n-bit Gray code sequence is a sequence of 2ⁿ integers where:

  • Every integer is in range [0, 2ⁿ − 1]
  • The first integer is 0
  • No integer appears more than once
  • Adjacent integers differ by exactly one bit
  • The first and last integers also differ by exactly one bit

Given n, return any valid n-bit Gray code sequence.
Example 1 — n = 2
Output: [0, 1, 3, 2]
Binary: 00 → 01 → 11 → 10 → (back to 00)
Each step flips exactly one bit ✓
Example 2 — n = 1
Output: [0, 1]
Example 3 — n = 3
Output: [0,1,3,2,6,7,5,4]
000→001→011→010→110→111→101→100

Constraints

1 ≤ n ≤ 16
Output must have exactly 2ⁿ integers
Any valid sequence is accepted
Complexity
Time: O(2ⁿ)
Space: O(2ⁿ) for the output
Must output all 2ⁿ values — can't do better than O(2ⁿ).
Key insight
This is NOT a search problem. The answer can be built directly by the reflection trick — no backtracking needed.

def grayCode(n: int) -> list[int]:
    result = [0]
    for i in range(n):
        # mirror current list in reverse
        result += [x | (1 << i) for x in reversed(result)]
    return result

def grayCode(n: int) -> list[int]:
    # Gray code of integer i = i XOR (i >> 1)
    return [i ^ (i >> 1) for i in range(1 << n)]

def grayCode(n: int) -> list[int]:
    if n == 0: return [0]
    prev = grayCode(n - 1)
    # prepend 0-bit to prev, then prepend 1-bit to reversed prev
    return prev + [x | (1 << (n-1)) for x in reversed(prev)]

Step 1 — What makes a sequence "Gray"?

Every adjacent pair must differ by exactly one bit. Including the wrap-around (last → first).
✓ Valid (n=2)
00 → 01  1 bit
01 → 11  1 bit
11 → 10  1 bit
10 → 00  1 bit ✓ (wrap)
✗ Invalid
00 → 11  2 bits ✗
00 → 01 → 10  01→10 = 2 bits ✗

Step 2 — The reflection trick

Start with [0]. At each step, mirror the current list in reverse and prepend the new bit (1) to the mirrored half. The boundary between the two halves always has exactly one bit change because the last element of the original and the first element of the mirror are the same number — just with the new bit added.
start: [0]
i=0:  [0] + mirror([0|1]) = [0, 1]
i=1:  [0,1] + mirror([1,0]|2) = [0,1, 3,2]
i=2:  [0,1,3,2] + mirror([2,3,1,0]|4) = [0,1,3,2, 6,7,5,4]
🔑 The join point (2→6, both = 010 and 110) always differs by exactly the one new bit — guaranteed by the mirror.

Step 3 — Why the XOR formula works

There's an elegant closed-form: gray(i) = i XOR (i >> 1). This converts any integer directly to its Gray code.
i=0: 000 XOR 000 = 000 = 0
i=1: 001 XOR 000 = 001 = 1
i=2: 010 XOR 001 = 011 = 3
i=3: 011 XOR 001 = 010 = 2
i=4: 100 XOR 010 = 110 = 6
The XOR formula and the reflection trick produce the same sequence — two ways to see the same structure.

Step 5 — Common mistakes

✗ Confusing n with 2ⁿ
n=4 means 4-bit codes, output has 2⁴=16 numbers. The loop runs n times, not 2ⁿ times.
✗ Forgetting to reverse
Appending forward instead of reversed breaks the 1-bit guarantee at the join point.
✗ Wrong shift in XOR formula
i ^ (i >> 1) not i ^ i >> 1 (operator precedence). Always use parentheses.
✗ Trying brute force / backtracking
This problem has a direct O(2ⁿ) construction. No need to search all permutations.

def grayCode(n):
  result = [0]
  for i in range(n):
    rev = reversed(result)
    for x in rev:
      result.append(x | 1<<i)
  return result
