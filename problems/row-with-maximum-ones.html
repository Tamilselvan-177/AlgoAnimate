Row With Maximum Ones

Scan each row of a binary matrix, count its ones, and track which row has the most. Ties go to the smaller row index.

⏱ ~5 min
Problem Statement
Given an m × n binary matrix mat, find the row with the maximum count of ones.

If multiple rows tie, pick the one with the smallest index. Return [rowIndex, count].
Example 1
mat = [[0,1],[1,0]]
Output: [0,1] — tie → smallest index wins
Example 2
mat = [[0,0,0],[0,1,1]]
Output: [1,2]
Example 3
mat = [[0,0],[1,1],[0,0]]
Output: [1,2]
Constraints
1 ≤ m, n ≤ 100
mat[i][j] is 0 or 1
Complexity
Time: O(m·n)
Space: O(1)
Key: one pass — count row, update best only if strictly greater.
sum() one-liner
Explicit loop
Pythonic one-liner
O(m·n)O(1)
def rowAndMaximumOnes(mat):
    best_idx, best_cnt = 0, 0
    for i, row in enumerate(mat):
        cnt = sum(row)                    # count ones in row
        if cnt > best_cnt:             # strictly greater → update
            best_idx, best_cnt = i, cnt
    return [best_idx, best_cnt]

def rowAndMaximumOnes(mat):
    best_idx, best_cnt = 0, 0
    for i in range(len(mat)):
        cnt = 0
        for v in mat[i]:
            cnt += v                      # accumulate
        if cnt > best_cnt:             # strictly greater only
            best_idx, best_cnt = i, cnt
    return [best_idx, best_cnt]
Step 1 — Count ones per row
Since values are only 0 or 1, summing a row gives its count of ones directly. No need for any conditional inside the inner loop.
cnt = sum([0, 1, 1, 0])  →  2
Step 2 — Strictly-greater update (tie rule)
✓ Correct — strictly >
if cnt > best_cnt:
Ties are ignored → first occurrence (smallest index) wins automatically.
✗ Wrong — >=
if cnt >= best_cnt:
This would overwrite with a later row on a tie, returning the last max row, not the first.

Step 4 — Common mistakes
✗ Using >= instead of >
Returns the last tied row, not the first. Always use strict greater-than.
✗ Initialising best_idx = -1
If no row ever beats 0 (all-zero matrix), you'd return -1. Start at idx=0, cnt=0.
✗ Returning max count globally
You need both the index AND the count — track them together, not separately.
✗ Sorting rows
Sorting loses row indices and is O(mn log m) — unnecessary. A single linear pass suffices.
Pattern: "running max with index" — the same template appears in Best Time to Buy Stock, Max Subarray, and dozens of scan problems. Track (best_val, best_idx), update only on strict improvement.

def rowAndMaximumOnes(mat):
  best_idx, best_cnt = 0, 0
  for i, row in enumerate(mat):
    cnt = sum(row)         # count ones
    if cnt > best_cnt:   # strictly greater
      best_idx = i
      best_cnt = cnt
  return [best_idx, best_cnt]
