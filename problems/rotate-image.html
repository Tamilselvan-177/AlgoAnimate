Rotate Image

Rotate an n×n matrix 90° clockwise in-place. The trick: two steps, no extra memory — transpose (flip across the diagonal), then reverse each row.

⏱ ~10 min to understand

Problem Statement

Given an n × n 2D matrix representing an image, rotate it by 90 degrees clockwise in-place.

You must modify the input matrix directly — do NOT allocate another 2D matrix.
Example 1 — n = 3
Input: [[1,2,3],[4,5,6],[7,8,9]]
Output: [[7,4,1],[8,5,2],[9,6,3]]
Example 2 — n = 4
Input: [[5,1,9,11],[2,4,8,10],[13,3,6,7],[15,14,12,16]]
Output: [[15,13,2,5],[14,3,4,1],[12,6,8,9],[16,7,10,11]]

Constraints

n == matrix.length == matrix[i].length
1 ≤ n ≤ 20
−1000 ≤ matrix[i][j] ≤ 1000
Must be in-place — O(1) extra space
Key insight
90° clockwise = transpose + reverse rows. Two O(n²) passes, zero extra memory.
Transpose + Reverse
4-way cycle
Transpose then reverse each row
O(n²)O(1)
from typing import List

def rotate(matrix: List[List[int]]) -> None:
    n = len(matrix)

    # Step 1: Transpose — swap matrix[i][j] with matrix[j][i]
    for i in range(n):
        for j in range(i + 1, n):
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

    # Step 2: Reverse each row
    for row in matrix:
        row.reverse()
Why this works
90° CW: result[j][n-1-i] = matrix[i][j]
Transpose: matrix[i][j] ↔ matrix[j][i] → gives matrix[j][i]
Reverse row: matrix[j][i] → matrix[j][n-1-i] ✓
4-way cycle — rotate 4 cells at a time
O(n²)O(1)
from typing import List

def rotate(matrix: List[List[int]]) -> None:
    n = len(matrix)
    for i in range(n // 2):
        for j in range(i, n - 1 - i):
            # save top
            tmp = matrix[i][j]
            # left → top
            matrix[i][j]         = matrix[n-1-j][i]
            # bottom → left
            matrix[n-1-j][i]     = matrix[n-1-i][n-1-j]
            # right → bottom
            matrix[n-1-i][n-1-j] = matrix[j][n-1-i]
            # top (saved) → right
            matrix[j][n-1-i]     = tmp
Only iterates the top-left quarter. Each step rotates 4 corresponding cells. Slightly more complex but elegant.

Step 1 — Where does each cell go after 90° CW?

For an n×n matrix, after rotating 90° clockwise, the element at [i][j] moves to [j][n−1−i]. Verify: corner [0][0] → [0][n−1] (top-left → top-right ✓).
Before (n=3)
[1,2,3] ← top row
[4,5,6]
[7,8,9] ← bottom row
After 90° CW
[7,4,1] ← was left col
[8,5,2]
[9,6,3] ← was right col

Step 2 — Transpose: flip across the main diagonal

Transpose swaps matrix[i][j] with matrix[j][i] for all j > i (upper triangle only, to avoid swapping back).

[4, 5, 6]
[7, 8, 9]

Start: [[1,2,3],[4,5,6],[7,8,9]]
Transpose swaps:
[0][1]↔[1][0]: 2↔4 → [[1,4,3],[2,5,6],[7,8,9]]
[0][2]↔[2][0]: 3↔7 → [[1,4,7],[2,5,6],[3,8,9]]
[1][2]↔[2][1]: 6↔8 → [[1,4,7],[2,5,8],[3,6,9]]
After transpose: [[1,4,7],[2,5,8],[3,6,9]]
Reverse rows:
Row 0: [1,4,7] → [7,4,1]
Row 1: [2,5,8] → [8,5,2]
Row 2: [3,6,9] → [9,6,3]
Result: [[7,4,1],[8,5,2],[9,6,3]] ✓

Step 5 — Common mistakes

✗ Transposing full square
Starting j at 0 instead of i+1 swaps each pair twice — undoing all changes. Always use upper triangle only.
✗ Wrong direction
90° counter-clockwise = transpose then reverse columns (or reverse rows first, then transpose). Don't mix them up.
✗ Allocating new matrix
Creating result = [[0]*n for _ in range(n)] violates the in-place constraint. The interviewer will catch it.
✗ Not squinting at 1×1
n=1: transpose does nothing, reverse does nothing → correct (single cell doesn't move). Always verify edge cases mentally.
The pattern: 90° CW = transpose + reverse rows. 90° CCW = transpose + reverse columns (or reverse rows first, then transpose). 180° = reverse the whole matrix. All are O(n²) time, O(1) space.
