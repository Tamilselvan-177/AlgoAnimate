Sliding Window Median

The median of a sorted list is the middle value; if the size is even, use the average of the two middles. Slide a window of size k across nums and return the median for each position (answers within 10⁻⁵ are accepted).

⏱ ~15 min to understand

Problem Statement

You are given an integer array nums and an integer k. There is a sliding window of size k that moves from the left of the array to the right; at each position you only see those k numbers. Return the array of medians, one per window, in order.
Example 1
nums = [1,3,-1,-3,5,3,6,7], k = 3
Output: [1.00000,-1.00000,-1.00000,3.00000,5.00000,6.00000]
Windows: [1,3,-1]→1, [3,-1,-3]→-1, [-1,-3,5]→-1, …
Explanation → Step 7 has a full window-by-window table for this array.
Example 2
nums = [1,2,3,4,2,3,1,4,2], k = 3
Output: [2.00000,3.00000,3.00000,3.00000,2.00000,3.00000,2.00000]

Constraints

1 ≤ k ≤ nums.length ≤ 10⁵
−2³¹ ≤ nums[i] ≤ 2³¹ − 1

Naive (sort each window): O(n · k log k)
Two heaps / multiset: O(n log k) time
Optimal — two heaps + lazy delete

small = max-heap of the lower half (Python: store -x). large = min-heap of the upper half. Track logical sizes with small_size / large_size. Values leaving the window go into delayed; when they reach a heap top, prune pops them. rebalance keeps small_size == large_size or small_size == large_size + 1 (median is always at small’s top for odd k, or average of both tops for even k).
from collections import defaultdict
from heapq import heappush, heappop
from typing import List


class MedianFinder:
    def __init__(self, k: int):
        self.k = k
        self.small = []  # max-heap (negated values)
        self.large = []  # min-heap
        self.delayed = defaultdict(int)
        self.small_size = 0
        self.large_size = 0

    def prune(self, pq):
        sign = -1 if pq is self.small else 1
        while pq and sign * pq[0] in self.delayed:
            self.delayed[sign * pq[0]] -= 1
            if self.delayed[sign * pq[0]] == 0:
                self.delayed.pop(sign * pq[0])
            heappop(pq)

    def rebalance(self):
        if self.small_size > self.large_size + 1:
            heappush(self.large, -heappop(self.small))
            self.small_size -= 1
            self.large_size += 1
            self.prune(self.small)
        elif self.small_size < self.large_size:
            heappush(self.small, -heappop(self.large))
            self.large_size -= 1
            self.small_size += 1
            self.prune(self.large)

    def add_num(self, num: int):
        if not self.small or num <= -self.small[0]:
            heappush(self.small, -num)
            self.small_size += 1
        else:
            heappush(self.large, num)
            self.large_size += 1
        self.rebalance()

    def remove_num(self, num: int):
        self.delayed[num] += 1
        if num <= -self.small[0]:
            self.small_size -= 1
            if num == -self.small[0]:
                self.prune(self.small)
        else:
            self.large_size -= 1
            if num == self.large[0]:
                self.prune(self.large)
        self.rebalance()

    def find_median(self) -> float:
        self.prune(self.small)
        self.prune(self.large)
        if self.k & 1:
            return float(-self.small[0])
        return (-self.small[0] + self.large[0]) / 2.0


class Solution:
    def medianSlidingWindow(self, nums: List[int], k: int) -> List[float]:
        finder = MedianFinder(k)
        for x in nums[:k]:
            finder.add_num(x)
        ans = [finder.find_median()]
        for i in range(k, len(nums)):
            finder.add_num(nums[i])
            finder.remove_num(nums[i - k])
            ans.append(finder.find_median())
        return ans

def medianSlidingWindow(nums, k):
    out = []
    for i in range(len(nums) - k + 1):
        w = sorted(nums[i : i + k])
        if k % 2 == 1:
            out.append(float(w[k // 2]))
        else:
            out.append((w[k // 2 - 1] + w[k // 2]) / 2.0)
    return out
💡 Same window motion as max-sum sliding window — drop the left element, add the right — but the median needs order statistics, so maintaining the window costs O(log k) per step with two heaps, not O(1).

Step 1 — What “median” means here

For each contiguous block of k values, sort them mentally. Odd k: the median is the single middle (sorted index k // 2). Even k: average the two middles at indices k//2 - 1 and k//2 (e.g. [1,2,3,4] → 2.5). Return floats; 10⁻⁵ error is allowed.
odd k → median = w[k//2]    even k → median = (w[k//2 - 1] + w[k//2]) / 2

Step 2 — Naive TLE vs optimal complexity

A sliding sum updates with one add and one subtract. The median is not additive: it depends on how all k numbers rank together.
❌ Sort each window
O(k log k) per stop × O(n) windows → O(n · k log k). Too slow for n ≤ 10⁵ (matches the Animation tab).
✅ Two heaps + lazy delete
Each slide: insert + lazy-remove in O(log k) amortized → O(n log k) time, O(k) space (Solution tab, optimal variant).

Step 3 — Split the window into two halves

Use a max-heap for the lower half and a min-heap for the upper half. Python’s heapq is a min-heap, so store negated values in small to simulate max-heap.
small (max-heap)
The “smaller” half. Heap top = largest on that side — the natural lower-middle candidate.
large (min-heap)
The “larger” half. Heap top = smallest on that side — the upper-middle candidate.
Invariant: every window value is in exactly one heap; all of small ≤ all of large. After rebalance, small_size == large_size or small_size == large_size + 1 (the extra element sits in small when k is odd).

Step 4 — Read the median from heap tops

k odd → median = -small[0]
k even → median = (-small[0] + large[0]) / 2
Call prune on both heaps in find_median so the tops are not values that already left the window (Step 5).

Step 5 — Lazy deletion and prune

heapq does not support “delete this value anywhere” in O(log k). When the window slides, mark the outgoing value in delayed and decrease the logical size of the half it belonged to; leave the stale copy inside the heap.
prune(heap): while the top is delayed, heappop and decrement delayed. When that value eventually surfaces at the top, it disappears for good. Total pushes and pops per element stay bounded, so the whole algorithm stays O(n log k).

Step 6 — Slide loop and helpers (matches Solution code)

for x in nums[:k]: add_num(x)
ans = [find_median()]
for i in range(k, len(nums)):
    add_num(nums[i])           # enter window on the right
    remove_num(nums[i - k])    # exit window on the left
    ans.append(find_median())
add_num — push toward small or large by comparing to -small[0], then rebalance.
remove_num — delayed[x] += 1, fix small_size / large_size, prune if x was at a top, then rebalance.
rebalance — move tops between heaps until the size rule holds; prune the heap you popped.
