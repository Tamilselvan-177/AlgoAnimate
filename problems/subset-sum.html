Subset Sum

Given an array of positive integers and a target sum — does any subset add up to exactly that sum? A pure true/false decision problem, solved beautifully with a 2D DP table.

⏱ ~15 min to understand

Problem Statement

Given an array of positive integers arr[] and a value sum, determine if there is a subset of arr[] with sum equal to the given sum.

Return true if such a subset exists, false otherwise. You don't need to find the subset — just decide if one exists.
Example 1 — true
arr[] = [3, 34, 4, 12, 5, 2], sum = 9
Output: true
4 + 3 + 2 = 9 ✓
Example 2 — false
arr[] = [3, 34, 4, 12, 5, 2], sum = 30
Output: false
No subset sums to 30
Example 3 — full array
arr[] = [1, 2, 3], sum = 6
Output: true
1 + 2 + 3 = 6, take everything

Constraints

1 ≤ arr.size() ≤ 200
1 ≤ arr[i] ≤ 200
1 ≤ sum ≤ 10⁴

Why not brute force?

Brute force tries all 2ⁿ subsets. For n=200 that's more atoms than in the universe.

DP approach: Build a table where dp[i][j] = "can the first i elements reach sum j?" Fill in O(n × sum) = O(200 × 10000) = 2M operations. Fast.

def isSubsetSum(arr, target):
    n = len(arr)
    # dp[i][j] = True if we can make sum j using first i elements
    dp = [[False] * (target + 1) for _ in range(n + 1)]

    # Base case: sum 0 is always achievable (empty subset)
    for i in range(n + 1):
        dp[i][0] = True

    for i in range(1, n + 1):
        for j in range(1, target + 1):
            # Option 1: exclude arr[i-1] — same as without it
            dp[i][j] = dp[i-1][j]
            # Option 2: include arr[i-1] if it fits
            if arr[i-1] <= j:
                dp[i][j] = dp[i][j] or dp[i-1][j - arr[i-1]]

    return dp[n][target]
1D Space-Optimized DP
O(n × sum)O(sum)
def isSubsetSum(arr, target):
    # dp[j] = can we make sum j using elements seen so far?
    dp = [False] * (target + 1)
    dp[0] = True   # sum 0 always achievable

    for num in arr:
        # iterate RIGHT TO LEFT to avoid reusing same element
        for j in range(target, num - 1, -1):
            dp[j] = dp[j] or dp[j - num]

    return dp[target]

# Why right-to-left? Prevents using arr[i] more than once.
# If we go left-to-right, dp[j-num] might already include arr[i].
Backtracking (Exponential — TLE for large inputs)
O(2ⁿ)O(n)
def isSubsetSum(arr, target):
    def bt(idx, remaining):
        if remaining == 0: return True   # found!
        if idx == len(arr) or remaining < 0: return False
        # include arr[idx] OR skip it
        return bt(idx+1, remaining-arr[idx]) or bt(idx+1, remaining)

    return bt(0, target)

# ⚠ O(2^n) — fine for tiny arrays, TLE for n=200.
# Use the DP solution for production code.

Step 1 — What dp[i][j] means

The DP table is the heart of the solution. Every cell answers one question precisely:
dp[i][j]
= True if we can form sum j using any subset of the first i elements
= False otherwise
Rows = "how many elements are we allowed to use (0 to n)". Columns = "what sum are we trying to reach (0 to target)".

The answer is dp[n][target] — can we reach the target using all n elements?

🎯 What does j mean? (The column index)

j is the current sum we are trying to build — not an array index, not a position. It is an actual numeric value ranging from 0 up to the target.

Step 2 — The recurrence: include or exclude

For each element arr[i-1] and each sum j, we have exactly two choices:
Option A — Exclude arr[i-1]
dp[i][j] = dp[i-1][j]
"Can we reach j without
using arr[i-1] at all?"
Option B — Include arr[i-1]
dp[i][j] = dp[i-1][j - arr[i-1]]
"Can we reach j - arr[i-1]
without using arr[i-1]?"
dp[i][j] = dp[i-1][j] OR (arr[i-1] ≤ j AND dp[i-1][j-arr[i-1]])

Step 3 — Base cases (the seeds)

dp[i][0] = True for all i
Sum = 0 is always reachable — just pick the empty subset. The entire first column is True.
dp[0][j] = False for j > 0
Using zero elements, we can't reach any positive sum. The first row (except dp[0][0]) is False.
These two base cases seed the entire table. Every other cell is derived from them via the recurrence.

Step 4 — Concrete trace: arr=[3,4,2], target=5

arr[i-1] → s=0 s=1 s=2 s=3 s=4 s=5
i=0 (none) T F F F F F
i=1 (use 3?) T F F T F F
i=2 (use 4?) T F F T T F
i=3 (use 2?) T F T T T T ✓
dp[3][5] = True because dp[2][3] = True (we can reach 3 with {3}), and 3 + 2 = 5. So the subset {3, 2} works.
The DP insight: We never need to enumerate subsets explicitly. Instead, we ask "is there any way to reach this sum using these elements?" — and each cell answers it using two previously computed cells. This overlapping subproblem structure is the essence of dynamic programming.
💡 DP = remembering results of smaller subproblems to avoid recomputation. The tree has 2ⁿ paths — the table has n×sum cells.
