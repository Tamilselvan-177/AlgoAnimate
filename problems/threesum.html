3Sum

Find all unique triplets in an array that sum to zero. The classic sort + two-pointer pattern — fix one element, then squeeze two pointers inward to find the complement.

⏱ ~10 min to understand

Problem Statement

Given an integer array nums, return all unique triplets [a,b,c] such that a+b+c == 0. The solution set must not contain duplicate triplets.
Example 1
Input: [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
-1+(-1)+2=0 and -1+0+1=0
Example 2
Input: [0,1,1]
Output: []
Example 3
Input: [0,0,0]
Output: [[0,0,0]]

Constraints

3 ≤ nums.length ≤ 3000
-10&sup5; ≤ nums[i] ≤ 10&sup5;
No duplicate triplets in output
Complexity
Time: O(n²)
Space: O(1) extra
Sort is O(n log n), dominated by O(n²)

def threeSum(nums):
    nums.sort()
    res = []
    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i - 1]: continue
        if nums[i] > 0: break
        L, R = i + 1, len(nums) - 1
        while L < R:
            total = nums[i] + nums[L] + nums[R]
            if total == 0:
                res.append([nums[i], nums[L], nums[R]])
                while L < R and nums[L] == nums[L+1]: L += 1
                while L < R and nums[R] == nums[R-1]: R -= 1
                L += 1; R -= 1
            elif total < 0: L += 1
            else: R -= 1
    return res

def threeSum(nums):
    n = len(nums); res = set()
    for i in range(n):
        for j in range(i+1, n):
            for k in range(j+1, n):
                if nums[i]+nums[j]+nums[k]==0:
                    res.add(tuple(sorted([nums[i],nums[j],nums[k]])))
    return [list(t) for t in res]
💡 3Sum = TwoSum on a sorted array, run n times. Fix one element i, then use two pointers L and R to find the pair that sums to −nums[i] in O(n).

Step 1 — Why sort first? (the key enabler)

Sorting does three things that make everything else possible:
🔶
Steerable search
sum<0 → L++. sum>0 → R--. Sorted order makes this deterministic.
👮
Adjacent duplicates
Equal values are neighbours. Skip with while nums[i]==nums[i-1].
✂
Early break
nums[i]>0 → no triple can sum to 0. Stop entirely.
Unsorted: [-1,0,1,2,-1,-4]   → sort →   [-4,-1,-1,0,1,2] duplicates adjacent, negatives first

Step 2 — The two-pointer squeeze and why it finds everything

For fixed nums[i], we need nums[L] + nums[R] = target = −nums[i]. Because the window [L…R] is sorted:
Decision at each (L, R) pair
nums[L]+nums[R] == target → record, skip dupes, L++, R--
nums[L]+nums[R] < target → too small → L++ (get a bigger left)
nums[L]+nums[R] > target → too big → R-- (get a smaller right)
Each pointer moves at most n steps. For each of n fixed values of i, the inner loop is O(n). Total = O(n²) vs O(n³) brute force.

Step 3 — Deduplication: the trickiest part

Without dedup, [-1,-1,0,1,1] would produce [-1,0,1] twice. We handle this at two independent levels:
Level 1 — Outer loop (i)
if i > 0 and nums[i] == nums[i-1]:
  continue

Skip if this anchor is the same value as the previous one. Run TwoSum only once per unique anchor.
Level 2 — After finding triplet
while nums[L]==nums[L+1]: L++
while nums[R]==nums[R-1]: R--

Skip past equal values on both sides before advancing.
Why “i > 0” in the outer guard matters i=0: always process (no previous element to compare)
i>0: skip only if same as previous → first occurrence runs, all copies skip

Step 4 — Full trace: [-4,-1,-1,0,1,2]

i=0 val=-4  L=1,R=5  sum=-3<0→L++ → sum=-2<0→L++ → sum=-1<0→L++ → L≥R done
i=1 val=-1  L=2,R=5  sum=0 ✓ → record [-1,-1,2] → L=3,R=4
L=3,R=4  sum=0 ✓ → record [-1,0,1] → L=4,R=3 done
i=2 val=-1 == nums[1] → skip (outer dedup)
i=3 val=0  L=4,R=5  sum=3>0→R-- → L≥R done
i=4 val=1 > 0 → break
Result: [[-1,-1,2], [-1,0,1]]

Step 5 — The kSum family: pattern generalisation

3Sum is one instance of a general pattern. Each level wraps the previous one:
Problem Approach Time Core idea
2Sum (sorted) Two pointers O(n) L+R squeeze on sorted array
3Sum (this) Fix i, run 2Sum O(n²) n iterations × O(n) = O(n²)
4Sum Fix i,j → run 2Sum O(n³) n² iterations × O(n) = O(n³)
Pattern: kSum = fix one element + (k-1)Sum on the rest. Sort once, dedup at every level.

Common mistakes that cause wrong answers

❌ Forgetting i > 0 guard
nums[i]==nums[i-1] at i=0 accesses index -1 — crash or wrong result.
❌ Not advancing after found
After dup-skipping, must do L++; R-- or the loop stalls on the same pair forever.
❌ Wrong loop bound
Need 2 elements after i for L and R. Use range(len(nums)-2) not range(len(nums)-1).
❌ Using a set to dedup output
Works but wastes O(n²) space. The in-place skip approach is O(1) extra and cleaner.
The 3Sum mental model: Sort → fix nums[i] → two-pointer TwoSum with target=−nums[i] on the right window → skip dupes at both levels → break when nums[i]>0.
