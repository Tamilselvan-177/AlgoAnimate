3Sum

Find all unique triplets in an array that sum to zero. The classic sort + two-pointer problem — fix one element, then squeeze two pointers inward to hunt for the complement.

Problem Statement

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i ≠ j, i ≠ k, j ≠ k, and nums[i] + nums[j] + nums[k] == 0.

The solution set must not contain duplicate triplets. Return in any order.
Example 1
Input: nums = [-1,0,1,2,-1,-4]
Output: [[-1,-1,2],[-1,0,1]]
-1+(-1)+2=0 and -1+0+1=0
Example 2
Input: nums = [0,1,1]
Output: []
No valid triplet sums to 0
Example 3
Input: nums = [0,0,0]
Output: [[0,0,0]]
0+0+0=0

Constraints & Key Facts

3 ≤ nums.length ≤ 3000
-10⁵ ≤ nums[i] ≤ 10⁵
Result must have no duplicate triplets

Space: O(1) extra

def threeSum(nums):
    nums.sort()            # sort ascending
    res = []

    for i in range(len(nums) - 2):
        # skip duplicate values for the fixed element
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        if nums[i] > 0:  # sorted: no triple can sum to 0
            break

        L, R = i + 1, len(nums) - 1
        while L < R:
            total = nums[i] + nums[L] + nums[R]
            if total == 0:
                res.append([nums[i], nums[L], nums[R]])
                # skip duplicates on both sides
                while L < R and nums[L] == nums[L+1]: L += 1
                while L < R and nums[R] == nums[R-1]: R -= 1
                L += 1; R -= 1
            elif total < 0:
                L += 1  # need bigger sum
            else:
                R -= 1  # need smaller sum

    return res

def threeSum(nums):
    nums.sort()
    res = []

    for i in range(len(nums) - 2):
        if i > 0 and nums[i] == nums[i-1]:
            continue
        seen = set()
        j = i + 1
        while j < len(nums):
            complement = -nums[i] - nums[j]
            if complement in seen:
                res.append([nums[i], complement, nums[j]])
                # skip duplicates
                while j + 1 < len(nums) and nums[j] == nums[j+1]:
                    j += 1
            seen.add(nums[j])
            j += 1

    return res

# For each fixed i, store seen elements in a hash set.
# When we encounter nums[j], check if its complement
# (-nums[i] - nums[j]) was already seen.

def threeSum(nums):
    n = len(nums)
    res = set()
    for i in range(n):
        for j in range(i + 1, n):
            for k in range(j + 1, n):
                if nums[i] + nums[j] + nums[k] == 0:
                    triplet = tuple(sorted([nums[i], nums[j], nums[k]]))
                    res.add(triplet)
    return [list(t) for t in res]

# TLE for n > ~200 but correct.
# Use a set of sorted tuples to avoid duplicates.

Step 1 — Why sorting unlocks everything

Sorting the array first gives us two superpowers: we can skip duplicates easily (same values are adjacent), and we can steer the two-pointer search by comparing the running total to zero.
📶
Sorted Order
-4,-1,-1,0,1,2
🎯
Pointer Direction
sum<0 → L++ / sum>0 → R--
If nums[i] > 0 and the array is sorted, we can break early — no three elements starting at a positive value can sum to zero.

Step 2 — Fix i, squeeze L and R

For each outer index i, we set L = i+1 and R = n-1. We then squeeze the window inward based on the current sum.
Trace: nums=[-4,-1,-1,0,1,2], i=1 (nums[i]=-1)
Array: -4 [-1] -1 0 1 [2] → L=2,R=5, sum=0 ✓ → record [-1,-1,2]
skip dup L, skip dup R → L=3, R=4
sum = -1+0+1 = 0 ✓ → record [-1,0,1]
L=4,R=3 → L≥R, inner loop done
i=2: nums[2]==nums[1] → skip (duplicate fixed element)
Result: [[-1,-1,2],[-1,0,1]]
The inner two-pointer loop runs at most O(n) for each fixed i, giving us O(n²) overall — far better than the O(n³) brute force.

Step 3 — Handling duplicates without a set

We deduplicate in two places: the outer loop (skip if nums[i] == nums[i-1]) and the inner loop after finding a triplet.
✓ Outer dedup
if i > 0 and nums[i] == nums[i-1]:
  continue
→ skip repeated fix point
✓ Inner dedup
while nums[L]==nums[L+1]: L++
while nums[R]==nums[R-1]: R--
→ skip repeated partners
Deduplication is O(1) per skip because we only move pointers forward — we never go back.
